UI개선포인트) 다음 항목들을 중점적으로 체크하여 개선:
1. 컴포넌트 간 간격(margin, padding)이 피그마와 정확히 일치하는지
2. 텍스트 정렬, 색상, 폰트 크기가 디자인과 일치하는지
3. 버튼 크기, 라운드, 색상이 정확한지
4. 선택/비선택 상태의 시각적 차이가 명확한지
5. 입력 필드 placeholder 색상, 테두리가 정확한지
6. 전체적인 비주얼 일관성 유지
7. 모바일 환경에서의 터치 영역 적절성

사전분석단계)
1. 기존 index.tsx 읽고 기능 로직 파악 (수정 금지 영역)
2. 기존 styles.ts 읽고 현재 스타일 확인
3. 피그마 디자인 다시 분석 (세부 수치 확인)
4. globals.css 색상/타이포그래피 토큰 재확인
5. 개선이 필요한 UI 요소 목록화

==============================================

구현단계)
1. index.tsx: JSX 구조는 유지하되 필요시 className/style 참조만 조정
2. styles.ts: 스타일 값만 정밀하게 수정 (크기, 간격, 색상 등)
3. 피그마 디자인과 1px 단위로 비교하며 수정
4. 토큰 기반 색상/타이포그래피 재확인
