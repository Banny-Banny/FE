핵심요구사항-시나리오)
1) 사용자가 "카카오페이로 결제하기" 버튼을 눌러 결제를 완료한다.
2) 결제 요청이 성공(성공 콜백 수신)한 직후 화면 상단/하단 어디서든 자연스럽게 "결제 완료 모달"이 나타난다.
3) 모달은 다음 정보를 명확하게 보여준다.
   - 결제 수단: "카카오페이"
   - 결제 상태: "결제가 완료되었어요" 또는 이와 유사한 긍정적인 문구
   - 결제 금액 요약: 최종 결제 금액 또는 간단한 주문 요약 문구
4) 사용자가 다음 행동을 선택할 수 있어야 한다.
   - 기본 액션: "확인" 또는 "완료" 버튼 (모달 닫기 및 다음 화면으로 이동)
   - 선택 액션(선택): "타임캡슐 보러가기", "홈으로 가기" 등 서비스 흐름에 맞는 추가 버튼 (필요 시)

핵심요구사항-UI구조)
1) 배경:
   - 화면 전체를 덮는 딤드(Dimmed) 오버레이
   - 터치 시 모달 바깥 탭으로 닫을지 여부는 기획에 따라 결정하되, 기본적으로 명시적 버튼(확인/완료)으로 닫는 패턴을 우선한다.
2) 모달 컨테이너:
   - 가운데 정렬된 카드형 모달 (Figma 디자인 기준 정렬/크기/모서리 반경을 그대로 적용)
   - 상단에는 결제 완료를 상징하는 아이콘 또는 체크마크(assets/icons 내 리소스 활용)
   - 중앙에는 타이틀/설명 텍스트
   - 하단에는 1개 이상 액션 버튼(Primary / Secondary 구분 가능)
3) 버튼:
   - Primary 버튼: "확인" 또는 "계속하기" 역할, 가장 강조되는 버튼
   - Secondary 버튼(선택): "홈으로 가기" 또는 "타임캡슐 목록으로" 등

핵심요구사항-동작관점)
1) 이 프롬프트에서는 "디자인/레이아웃"에 집중하고, 실제 카카오페이 결제 로직은 구현하지 않는다.
2) 결제 성공 여부는 상위에서 props로 내려주는 boolean 또는 상태값으로 가정하고,
   - 예: isVisible, isPaymentSuccess 등의 props를 통해 모달 노출 여부를 제어하는 형태로 설계
3) 모달 내 버튼 클릭 시 호출될 콜백(onClose, onConfirm 등)만 props로 정의하고, 내부에서는 UI만 구현한다.

구현단계)
1) index.tsx
   - React Native 기반의 `Modal`, `View`, `Text`, `TouchableOpacity` 등을 활용하여 Figma 구조를 1:1로 옮길 것.
   - props 예시 (실제 타입은 프로젝트 상황에 맞게 조정):
     - isVisible: boolean (모달 노출 여부)
     - amount: number (결제 금액)
     - paymentMethod: 'kakaopay' | string (결제 수단, 기본값은 카카오페이)
     - onConfirm: () => void (확인 버튼 클릭 시 호출)
     - onSecondaryAction?: () => void (선택 액션 버튼 클릭 시 호출)
   - 시맨틱한 컴포넌트/변수 이름 사용 (예: `PaymentCompleteModal`, `titleText`, `descriptionText`, `primaryButtonText` 등)

2) styles.ts
   - `StyleSheet.create()`로 모든 스타일을 정의
   - Figma에서 추출한 값(여백, 패딩, 정렬, 모서리 반경, 그림자, 폰트 사이즈 등)을 최대한 그대로 반영
   - 색상은 모두 globals.css의 토큰을 참고하여 매핑 (예: 배경 딤드 색상, 카드 배경 색상, 버튼 색상 등)
   - 텍스트 스타일은 타이틀/본문/버튼 텍스트 등 역할에 따라 분리

3) 접근성/사용성
   - 모달 오버레이 영역과 모달 카드 영역을 명확히 구분
   - 버튼에 `accessibilityRole="button"` 및 적절한 `accessibilityLabel`을 부여
