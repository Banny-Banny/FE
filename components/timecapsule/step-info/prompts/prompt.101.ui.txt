구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @.cursor/rules/01-common.mdc
            - @.cursor/rules/02-wireframe.mdc
            - @.cursor/rules/03-ui.mdc
            - @.cursor/rules/figma.mdc

==============================================

조건-피그마) 활용할MCP: CursorTalkToFigmaMCP(채널명: 여기에_채널명_입력)
조건-피그마) 구현될 컴포넌트 노드ID: 여기에_노드ID_입력

조건-파일경로) 구현될 TSX 파일경로: components/timecapsule/step-info/index.tsx
조건-파일경로) 구현될 CSS 파일경로: components/timecapsule/step-info/styles.ts

==============================================

조건-스타일시스템) React Native StyleSheet 사용할 것
조건-스타일시스템) StyleSheet.create() 방식으로 styles.ts 작성
조건-스타일시스템) 인라인 스타일 금지 (style={{...}} 금지)
조건-스타일시스템) 모든 스타일은 styles.ts에서만 선언하고 index.tsx에서 import하여 사용

==============================================

조건-공통컴포넌트) 공통 컴포넌트를 최대한 활용할 것
조건-공통컴포넌트) 컴포넌트 경로: components/

조건-색상토큰) globals.css에 명시된 색상 변수 토큰 활용
조건-색상토큰) 비슷한 토큰 선택하여 사용 (하드코딩 금지)
조건-색상토큰) hex, rgb, hsl 직접 입력 금지

조건-타이포그래피) globals.css에 명시된 타이포그래피 변수 토큰 활용
조건-타이포그래피) 비슷한 토큰 선택하여 사용 (하드코딩 금지)

==============================================

조건-아이콘/이미지) 아이콘 경로: assets/icons/
조건-아이콘/이미지) 이미지 경로: assets/images/
조건-아이콘/이미지) 외부 아이콘 패키지 설치 금지
조건-아이콘/이미지) 제공된 assets 폴더 내 리소스만 사용

==============================================

핵심요구사항) MCP를 연동하여 피그마의 디자인을 그대로 구현할 것.
핵심요구사항) 피그마에 표시된 사이즈, 간격, 색상 그대로 적용
핵심요구사항) 추가적인 기능이나 애니메이션 넣지 말고 디자인만 완벽히 구현

==============================================

사전분석단계)
1. package.json 확인하여 사용 가능한 라이브러리 분석
2. 피그마 구조 전체 분석 (레이아웃 → 섹션 → 컴포넌트 순)
3. globals.css의 색상/타이포그래피 토큰 목록 확인
4. 공통 컴포넌트 목록 확인 및 재사용 가능 여부 판단

==============================================

구현단계)
1. index.tsx: JSX 구조만 작성 (View, Text 등 기본 컴포넌트 사용)
2. styles.ts: StyleSheet.create()로 모든 스타일 선언
3. 피그마 디자인과 1:1 대응되도록 레이아웃 구현
4. 토큰 기반 색상/타이포그래피만 사용
